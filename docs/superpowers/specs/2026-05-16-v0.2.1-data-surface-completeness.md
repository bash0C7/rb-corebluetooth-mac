# v0.2.1 Spec — Library Data-Surface Completeness

**Date**: 2026-05-16
**Scope**: rb-corebluetooth-mac
**Driver**: User audit. Library is currently dropping data that CoreBluetooth delegates / APIs provide, masquerading as a thin wrapper while silently truncating the surface. This is a generic BLE library bug, not an enhancement.

**Verification source**: `/Library/Developer/CommandLineTools/SDKs/MacOSX.sdk/System/Library/Frameworks/CoreBluetooth.framework/Versions/A/Headers/*.h`. Every API / constant cited below was read from headers in this same session — not memory.

---

## 1. Goal

Surface every CoreBluetooth datum that the library can plausibly relay from `CBCentralManager` / `CBPeripheral` / their delegates / `CBCharacteristic` / `CBService` / `CBDescriptor` / `CBError` / `CBATTError`. After v0.2.1, a user who reads CoreBluetooth's public docs should not find a documented field that this library throws away.

**Non-goals**:
- L2CAP CoC (`CBL2CAPChannel`) — separate phase, BLE-spec heavy
- `willRestoreState:` — iOS background restoration semantics; macOS impact minimal
- `CBConnectPeripheralOption*` keys on `connect(_:options:)` — defer to v0.3
- Pairing/bonding programmatic control — CoreBluetooth auto-handles

## 2. Currently Omitted Data (Itemized)

### §1. Advertisement data

**Source**: `centralManager(_:didDiscover:advertisementData:rssi:)`'s `advertisementData: [String: Any]`.
**Keys** (from `CBAdvertisementData.h`):

| Key | Type | Notes |
|---|---|---|
| `CBAdvertisementDataLocalNameKey` | `String` | already partly used (fallback when `peripheral.name` nil) |
| `CBAdvertisementDataTxPowerLevelKey` | `NSNumber` (Int) | dBm |
| `CBAdvertisementDataServiceUUIDsKey` | `[CBUUID]` | advertised service UUIDs |
| `CBAdvertisementDataServiceDataKey` | `[CBUUID: Data]` | per-service ad data |
| `CBAdvertisementDataManufacturerDataKey` | `Data` | first 2 bytes = company ID (LE) |
| `CBAdvertisementDataOverflowServiceUUIDsKey` | `[CBUUID]` | iOS overflow area; macOS may surface or not |
| `CBAdvertisementDataIsConnectable` | `NSNumber` (Bool) | connectable in this instant |
| `CBAdvertisementDataSolicitedServiceUUIDsKey` | `[CBUUID]` | services the peripheral solicits |

### §2. Characteristic properties — full bitmask

**Source**: `CBCharacteristic.h:CBCharacteristicProperties`. Currently only 5 of 10 bits surfaced.

Missing: `broadcast` (0x01), `authenticatedSignedWrites` (0x40), `extendedProperties` (0x80), `notifyEncryptionRequired` (0x100), `indicateEncryptionRequired` (0x200).

### §3. Service `isPrimary` / `includedServices`

**Source**: `CBService.h`.
- `isPrimary: Bool` — readable from `CBService` directly after discovery
- `includedServices: [CBService]?` — **requires** explicit `discoverIncludedServices(_:for:)` call (per `CBPeripheral.h:144`). Not populated automatically.

### §4. Disconnect error

**Source**: `centralManager(_:didDisconnectPeripheral:error:)` (`CBCentralManager.h:346`). Currently not implemented at all.

### §5. Structured error code surface

**Source**: `CBError.h`. Two error domains:
- `CBErrorDomain` (`CBError` enum, codes 0..18) — peripheral / connection / encryption / pairing errors
- `CBATTErrorDomain` (`CBATTError` enum, codes 0x00..0x11) — GATT protocol errors

Currently `localizedDescription` is the only thing kept, code is discarded.

### §6. `maximumWriteValueLengthForType:`

**Source**: `CBPeripheral.h:177`. `(NSUInteger)maximumWriteValueLengthForType:(CBCharacteristicWriteType)type`. Tells the user how big a single chunk can be for `.withResponse` (typically 512) vs `.withoutResponse` (MTU-3, usually 20..512). Not exposed.

### §7. `readRSSI` after connect

**Source**: `CBPeripheral.h:119` `- (void)readRSSI;` + delegate `peripheral(_:didReadRSSI:error:)` (`CBPeripheral.h:317`). Currently no Ruby-side API. Scan-time RSSI is the only RSSI ever surfaced.

### §8. Descriptors API

**Source**: `CBPeripheral.h:225` `discoverDescriptorsForCharacteristic:`, `:234` `readValueForDescriptor:`, `writeValue:forDescriptor:`, delegates at `:398`, `:409`, `:420`. Currently completely missing. Standard descriptors (CCCD 0x2902, User Description 0x2901, Characteristic Format 0x2904, etc.) are unreachable.

### §9. `peripheralDidUpdateName` + `didModifyServices`

**Source**: `CBPeripheral.h:282` `peripheralDidUpdateName:`, `:294` `didModifyServices:`. Both delegate hooks unimplemented. Runtime name change / service-set invalidation invisible to users.

### §10. Discover-time characteristic value

**Source**: `CBCharacteristic.value` is optionally populated immediately after `didDiscoverCharacteristicsForService:` if the peripheral pre-cached it. Currently dropped — Ruby `Characteristic` exposes no `initial_value`.

### §11. `discoverServices` UUID filter

**Source**: `CBPeripheral.h` `- (void)discoverServices:(nullable NSArray<CBUUID *> *)serviceUUIDs;`. Library passes `nil` unconditionally (`CBMCentral.swift:168`). User cannot ask for "only GAP" or "only the NUS service" — gets entire GATT tree every time.

### §12. `Central#close` full implementation (folds in handoff L6)

**Source**: handoff `2026-05-16-v0.2.0-bugfix-handoff.md` §4 L6. `Central#close` is currently a no-op (`lib/corebluetooth_mac/central.rb:40-43`). Resources accumulate in `delegatesLock` / `knownPeripherals` / `CBMSubscriptionRegistry` without explicit teardown. Including in v0.2.1 because the new APIs (descriptors, peripheral-event hooks, additional locks) compound the cleanup surface.

## 4. API Shape (Ruby surface)

### 4.1 `DiscoveredDevice` — §1

Replace shape (no compat with v0.2.0):

```ruby
DiscoveredDevice = Data.define(
  :identifier,
  :name,
  :rssi,
  :tx_power_level,           # Integer (dBm), or nil
  :connectable,              # true | false | nil (key absent)
  :service_uuids,            # [String] lowercase UUID, default []
  :service_data,             # { String => String (binary) }, default {}
  :manufacturer_data,        # String (binary, Encoding::ASCII_8BIT), or nil
  :solicited_service_uuids,  # [String], default []
  :overflow_service_uuids,   # [String], default []
  :central_id,
)
```

JSON across C boundary: binary `Data` encoded as **hex string**; Ruby side decodes to `String` with `Encoding::ASCII_8BIT`. Service-data dict keys are UUID strings lowercased.

### 4.2 `Characteristic` — §2, §10

```ruby
attr_reader :uuid, :properties, :service, :initial_value

def supports?(prop) = @properties.include?(prop)
```

`@properties`: `Set<Symbol>` with full bitmask coverage:
`:broadcast`, `:read`, `:write_without_response`, `:write`, `:notify`, `:indicate`,
`:authenticated_signed_writes`, `:extended_properties`,
`:notify_encryption_required`, `:indicate_encryption_required`.

**Existing `readable?` / `writable?` / `notifiable?` predicates are removed** — user code rewrites to:
- `ch.supports?(:read)` (instead of `readable?`)
- `ch.supports?(:write) || ch.supports?(:write_without_response)` (instead of `writable?`)
- `ch.supports?(:notify) || ch.supports?(:indicate)` (instead of `notifiable?`)

`@initial_value`: `String` (`Encoding::ASCII_8BIT`) or `nil`. Populated when `didDiscoverCharacteristicsForService:` delivers a pre-cached value.

### 4.3 `Service` — §3

```ruby
attr_reader :uuid, :peripheral, :characteristics, :is_primary, :included_service_uuids

def discover_included_services(timeout: 5.0)
  # returns [Service] objects newly populated for includedServices
end
```

`is_primary` is a `Boolean` from JSON, **direct attribute** (no `primary?` predicate).
`included_service_uuids` is `[]` until `discover_included_services` is called; then a `[String]` of lowercase UUIDs.

### 4.4 `Peripheral` — §4, §6, §7, §9, §11

```ruby
def discover_services(services: nil, timeout: 5.0)
  # services: nil | String | [String]   (UUID filter, nil = all)
end

def max_write_length(response: true)
  # Integer; calls cbm_peripheral_max_write_length(identifier, response_int)
end

def read_rssi(timeout: 5.0)
  # Integer (dBm)
end

def poll_events(timeout: 0.0)
  # returns next pending PeripheralEvent or nil
  # PeripheralEvent = NameUpdated(new_name:) | ServicesInvalidated(uuids:) | Disconnected(error:)
end

def last_disconnect_error
  # CoreBluetoothMac::CBError | CoreBluetoothMac::ATTError | nil
end
```

`PeripheralEvent` is a sealed value-object hierarchy (similar to `Subscription`'s data-define style): each subtype is a `Data.define`, all Ractor-shareable.

### 4.5 `Descriptor` — §8

```ruby
class Descriptor
  attr_reader :uuid, :characteristic, :value
  def read(timeout: 5.0); end       # returns Data
  def write(data, timeout: 5.0); end # returns true
end

# on Characteristic:
def discover_descriptors(timeout: 5.0) # returns [Descriptor]
def descriptors                         # [Descriptor], or raises if discover not called
```

### 4.6 Error class — §5

**Single `Error` class**. Existing `TimeoutError` / `ClosedError` / `ConnectionError` / `DiscoveryError` subclasses are **removed**.

```ruby
module CoreBluetoothMac
  class Error < StandardError
    attr_reader :domain, :code, :code_name
    # domain ∈ {:timeout, :closed, :connection, :discovery, :validation, :cb, :att}
    # code: Integer (only for :cb / :att), nil otherwise
    # code_name: Symbol (only for :cb / :att), nil otherwise
    def initialize(message, domain:, code: nil, code_name: nil)
      super(message)
      @domain, @code, @code_name = domain, code, code_name
    end
  end
end
```

User rescue pattern:
```ruby
rescue CoreBluetoothMac::Error => e
  case e.domain
  when :timeout    then ...
  when :cb         then puts "CB error code=#{e.code} (#{e.code_name})"
  when :att        then puts "ATT error code=#{e.code} (#{e.code_name})"
  end
end
```

Lookup tables (`code → code_name` symbol) for both `:cb` and `:att` domains live in `lib/corebluetooth_mac/error_codes.rb`. Generated from SDK header enum names: lowercase snake_case, prefix stripped (`CBErrorConnectionTimeout` → `:connection_timeout`, `CBATTErrorReadNotPermitted` → `:read_not_permitted`).

JSON wire format from Swift on error:
```json
{
  "ok": false,
  "error": {
    "domain": "cb" | "att" | "lib",
    "code": 6,
    "code_name": "connection_timeout",
    "message": "Connection attempt timed out"
  }
}
```
`"lib"` = library-internal (`code` / `code_name` null). Ruby C bridge maps `"lib"` JSON to one of `:timeout` / `:closed` / `:validation` / `:connection` / `:discovery` symbols based on the message tag prefix (e.g. `[timeout]`, `[closed]`).

### 4.7 `Central` — §12

```ruby
def close
  @native.close  # new C-bridged native call
  nil
end
```

Swift `CBMCentral.close()`:
1. Drain `CBMSubscriptionRegistry` for all entries `where central === self`
2. Iterate `delegatesLock` snapshot, set each peripheral's `delegate = nil` (must be on `queue.sync`)
3. Empty `delegatesLock` dict
4. Empty `knownPeripherals` dict
5. Set `manager.delegate = nil` (same fix as `cbm_central_free`)

`Central#close` becomes idempotent; subsequent `scan` / `connect` raise `ClosedError`.

## 7. Compatibility Policy

**No compat with v0.2.0**. v0.2.1 is a full API rewrite of the omitted-data surface. User code from v0.2.0 must be migrated:

- `Characteristic#readable?` / `#writable?` / `#notifiable?` removed → use `#supports?(:read)` etc.
- `Service#primary?` removed → use `#is_primary` attribute
- `TimeoutError` / `ClosedError` / `ConnectionError` / `DiscoveryError` removed → single `Error` class, rescue by `#domain`
- Native errors no longer wrapped in `RuntimeError`-ish form → all carry `#domain` / `#code` / `#code_name`
- `DiscoveredDevice` `Data.define` shape changes; member order reflowed (logical: identity → signal → ad-data → central)
- `Peripheral#discover_services` gains `services:` kwarg (default nil keeps old "discover all" behavior, but signature changed)
- JSON wire format internal — no user visibility

README's "Upgrading from v0.2.0" section enumerates these one-to-one. No shims.

## 9. Risks

1. **Descriptors require service-tree being fresh**: CCCD on a notify characteristic doesn't exist until `setNotifyValue(true:for:)` is called. Document this in `Characteristic#discover_descriptors` ruby docstring.
2. **`poll_events` event queue unbounded**: if user never polls, queue grows. Mitigation: cap at 256 events, drop oldest with a warning to stderr.
3. **Hex encoding overhead**: manufacturer data + service data are hex-encoded across JSON. For typical advertisements (<31 bytes) this is negligible; for very large service data, consider future binary side-channel (out of v0.2.1 scope).
4. **`includedServices` traversal can recurse**: spec only discovers one level via explicit `Service#discover_included_services`. Documented; not auto-traversed.
5. **`maximumWriteValueLengthForType:` is a property snapshot** — value can change post-MTU-negotiation. User must call after connect completion, not before. Documented in method docstring.
6. **`willRestoreState` not implemented**: macOS app suspension can lose state. Explicitly out of scope; user must not rely on cross-launch peripheral identity beyond `peripheral.identifier`.
